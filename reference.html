What are the reference data type ?

Reference (complex) type : A reference value is an instance of a specific reference type. In ECMAScript, reference types are structures used to group data and functionality together and are often incorrectly called "classes". Reference types are also sometimes called object definitions because they describe the properties and methods that objects should have. In other words, The reference type are collection of the primitive data (number, string, boolean, null and undefined) type.The reference type consists of objects , arrays (which are also objects),functions (despite they are not objects), classes, maps and sets.It is important to remember that primitive data type are immutable and reference data type are mutable.

What is the deference between primitive data type and reference data type ?

Primitive data type are immutable and reference data type are mutable. Mutable objects are objects whose values can change once created, while immutable objects are those whose values cannot change once created. So when we modify the primitive type (string for example) value , a new string is created and the string name assigned to the new value, The same goes for all primitive type. But, with reference data type it is the opposite exactly , because , they have individual identities the can be modified , manipulated and applied methods on them.

Why it is important to distinguish between primitive and reference data types ?

Consider you have multiple threads working on the same object. You cannot predict the outcome of your code since the action of one thread will modify the object, the next thread using this object as a parameter will get totally different data and there’s no guarantee that the operation will be safe. You cannot keep a copy of an object by assigning it to a variable first, then modifying the properties of the original object. The property values in the copy will also change. With immutable types, we know that a value’s state won’t be changed once we create it hence not resulting in unexpected logic in our code. For instance, we want to compare two email strings to authenticate a user. If a string was mutable and a part of our code had altered the email string inside some other method, then the user wouldn’t be able to log in.

The typeof Operator

You can use the JavaScript typeof operator to find the type of a JavaScript variable.The typeof operator returns the type of a variable or an expression.In addition , it is important to know that In JavaScript, a variable without a value, has the value undefined. The type is also undefined.

Objects in JavaScript

In real life , any object has properties and methods , for instance , a car is an object and it has properties like weight and color, and methods like start and stop, the same goes for objects in JavaScript , they also have properties and methods , you can consider an object like a variable with many values, remember that objects are mutable so you can easy remove or add value(s) because it is possible to modify objects. You can declare an object exactly like any other variable using let or const then a curly brackets and inside the curly brackets , you can put the object properties as key:value pair separated by comma, .You can access object properties in two ways the dot operator "." or by typing the property name inside square brackets objectName["propertyName"]. As for object methods ,they are actions that can be performed on objects.You can access the object methods using the dot operator ".". Note that the methods are ended with parentheses ( ).

Arrays in JavaScript

JavaScript arrays are used to store multiple values in a single variable , just like objects ,in fact arrays are objects. An array can hold many values under a single name, and you can access the values by referring to an index number [i].You can easily loops throw an array values and modify them (remember that arrays are objects).You can declare an array just like an object but use the square brackets [ ] instead of curly brackets.You can access an array element by referring to the index number [i] (note : the indexes start from 0). Arrays have properties and methods also but the deference between arrays and objects is arrays use numbers (indexes) to access its "elements", but Objects use names to access its "members". Ok so the question here is when to use arrays and when to use objects ? Well it is easy, you should use objects when you want the element names to be strings (text) and you should use arrays when you want the element names to be numbers.

Functions in JavaScript

A JavaScript function is a block of code designed to perform a particular task and it is executed when we (calls it).To declare a function all you have to do is type the name of the function followed by parentheses ( ) and then and arrow => and curly brackets { }.The parentheses may include parameter names separated by commas, (parameter1, parameter2, ...) and The code to be executed, by the function, is placed inside curly brackets { } ( note: Function parameters called Function arguments when the values received by the function when we called it). Ok, Why functions are important ? because you can define the code once, and use it many times.it is important to remember that, Variables declared within a JavaScript function, become local to the function.Local variables can only be accessed within the function.
